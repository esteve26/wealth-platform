# ARCHITECTURE_CONTEXT

This project is currently in the domain and architecture discovery phase.

No implementation stack has been finalized yet.

## Architectural Direction

The project should be designed using:

- Domain-Driven Design principles;
- explicit business capabilities;
- strong separation between domain concepts and technical implementation;
- deterministic financial calculation engines;
- AI as an explanatory and assistant layer, not as the source of financial truth.

## Key Architecture Principles

1. Business model first, technology second.
2. Financial truth is calculated by deterministic engines.
3. AI explains, summarizes and assists.
4. Transactions are immutable.
5. Assets are canonical master data.
6. Provider data is external evidence, not domain truth.
7. Historical traceability is mandatory.
8. Sensitive operations require clear permissions and auditability.
